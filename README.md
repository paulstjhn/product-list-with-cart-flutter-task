# Product List with Cart — Flutter Edition

**Flutter Mentorship Program — Final Project Brief**

You must've already built this once in JavaScript. You know the features, the user flows, and what the end result should feel like. Now you're going to build it again — in Flutter.

This isn't about learning what to build. It's about proving you understand how to build it the Flutter way: clean widget structure, proper state management with BLoC, and code that a teammate could actually read and maintain.

## 🚀 Getting Started

1. **Fork this repository** to your own GitHub account (button in the top right of the repo page)
2. **Clone your fork** locally:
   ```bash
   git clone git@github.com:YOUR-USERNAME/flutter-product-list-cart.git
   cd flutter-product-list-cart
   ```
3. Read [`docs/00-getting-started.md`](docs/00-getting-started.md) for the full setup walkthrough — initialising your Flutter project inside this repo, wiring up the `assets/images/` folder, and your first commit
4. Read [`docs/01-overview.md`](docs/01-overview.md), then decide which tier you're attempting and read the matching doc
5. Use [`CHECKLIST.md`](CHECKLIST.md) to track your progress as you build

---

## 🧭 Choosing Your Tier

**Tier 1 (Base)** is required for everyone. It's a fully functional product list and cart — local data, BLoC state management, and clean widget decomposition. See [`docs/02-tier-1-base.md`](docs/02-tier-1-base.md).

**Tier 2 (Intermediate)** is optional and builds on Tier 1 — responsive layout, UX polish (animations, transitions, empty states), and a Repository layer between your data source and BLoC. It adds 50 points on top of your Tier 1 score. See [`docs/03-tier-2-intermediate.md`](docs/03-tier-2-intermediate.md).

If you're unsure which to attempt, build Tier 1 completely and solidly first. A clean Tier 1 submission outscores a rushed, broken attempt at Tier 2.
