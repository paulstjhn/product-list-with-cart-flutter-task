# Ground Rules

Read this before you write a single line of code. Violating any of these can cap your grade regardless of how complete your feature set is.

- **You may reference the original JS project for feature reference, not for logic** — look at it to confirm what a feature should do, not to copy how it was implemented
- **You may use `flutter_bloc`, `equatable`, and any standard Flutter packages**
- **You may not use state management packages that abstract away BLoC** (e.g. GetX, Provider for cart state) — the point is to demonstrate BLoC
- **External UI kits are not allowed** — build your own widgets
- **AI-generated code is permitted as a reference tool**, but you must be able to explain every line in your verbal walkthrough
- **Late submissions lose 5 points per day**, up to a maximum of 15 points
